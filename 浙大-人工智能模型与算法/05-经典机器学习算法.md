## 聚类与降维

### K-Means 聚类

#### 一句话

> 把一堆数据点自动分成 K 个堆，每个堆里的点尽可能"像"，不同堆之间尽可能"不像"。

#### 核心目标

$$最小化 \sum(\text{每个点到它所属中心点的距离})$$

> 💭 **我的理解：** 比如总共 15 个点，K-Means 先选 K 个中心点（K 是自己定的），分成 K 类，然后目标是**所有点到各自中心点的距离总和最小**。

#### 算法步骤

```
随机选 K 个中心点 → 分配（每个点归最近中心）→ 移动（重新算平均值，中心点移过去）→ 重复直到稳定
```

> 💭 **我的理解：** K-Means 本质上就是"物以类聚"——近的归一堆，反复调整堆的边界，直到稳定。

#### 具体例子

6 个学生的成绩，分成 2 组（K=2）：

```
A: 数学 2分, 英语 2分    B: 数学 3分, 英语 2分
C: 数学 8分, 英语 8分    D: 数学 9分, 英语 8分
E: 数学 2分, 英语 1分    F: 数学 8分, 英语 9分
```

画在坐标上明显看出两个群体——左边低分、右边高分，K-Means 就是让机器自动找出来。

| 步骤 | 操作 |
|------|------|
| 1. 选中心 | 随机挑 A(2,2) 和 D(9,8) 作为初始中心点 |
| 2. 分配 | 算距离，近的归谁 → 组1: A,B,E 组2: C,D,F |
| 3. 移动 | 算每组平均值 → 组1 中心 (2.33, 1.67)，组2 中心 (8.33, 8.33) |
| 4. 重复 | 再分配、再移动...直到中心点不再变 |

#### 什么是"距离"

二维例子用的是画在纸上的直线距离（欧几里得距离）。但 K-Means 不限于二维：

| 场景 | 维度 | 每个维度是什么 |
|------|------|-------------|
| 学生分组 | 2维 | (数学分, 英语分) |
| 用户分群 | 3维 | (年龄, 月消费, 登录次数) |
| 图像压缩 | 几百维 | 每个像素是一个维度 |

公式本质一样——不管多少维，就是求高维空间的直线距离：

```
二维：  √[(x₁-x₂)² + (y₁-y₂)²]
n维：  √[(第1维差)² + (第2维差)² + ... + (第n维差)²]
```

> 💭 不管多少维度，思路都一样——在高维空间里量直线距离。

#### 关键特点

| 优点 | 缺点 |
|------|------|
| 简单直观、速度快 | 必须提前指定 K 是多少 |
| 适合大规模数据 | 对初始中心点敏感 |
| 适合球形分布 | 非球形分布效果差 |

> 💭 **关于"非全局最优"：** 第一步随机选中心点，运气差可能收敛到不够好的结果。实际应用会多跑几次（scikit-learn 默认 `n_init=10`），选最好的那次。

#### K-Means 与深度学习的配合

> 💭 **我的理解：** K-Means 帮深度学习快速摸清数据中的类别规律——比如一堆数据里大概有几类、哪些样本比较像。

**实际用途：**

| 用途 | 说明 |
|------|------|
| 数据探索 | 跑一遍 K-Means，了解数据里大概有几类，哪些样本聚在一起 |
| 快速打标签 | 聚类后手动给每组标类别名，比一张一张标注快得多 |
| 目标检测 | YOLO 用 K-Means 对标注框宽高聚类，确定锚框尺寸 |
| 特征分析 | 训练好之后对神经网络某层的输出跑 K-Means，看看模型学到了什么规律 |

**但要注意：**

> K-Means 找的是**纯数学上的距离近**，不等于语义类别。

比如人脸照片跑 K-Means，它可能按背景色分组（蓝天 vs 暗光），而不是按男女分——因为它只看像素数值，不懂内容含义。

> 💭 所以 K-Means 是**数据探索的工具**，帮你在深度学习之前或之后快速理解数据规律，但它不替代深度学习，也不能当分类器直接用。
