# 02 | 搜索与问题求解

> 参考教材：《人工智能引论》吴飞、潘云鹤

---

## 全景概览

本章讲了**两大类搜索**：

| 类型 | 核心问题 | 代表算法 |
|------|----------|----------|
| 启发式搜索 | 如何高效找到从起点到终点的最短路径 | 贪心最佳优先、Dijkstra、A* |
| 对抗搜索 | 在双方博弈中，如何做出最优决策 | Minimax → α-β 剪枝 → MCTS |

两者之间的关系：**搜索 = 在可能性空间中找最优解**，启发式搜索是单方寻路，对抗搜索是双方博弈，思路相通。

---

## 一、启发式搜索

### 1.1 贪心最佳优先搜索

- 评价函数：**f(n) = h(n)** — 只看当前节点到目标的预估代价
- 朝着目标方向走，效率高，但**不保证最优解**

> 💭 **我的理解：** 相当于导航只告诉你"终点在哪个方向"，你闷头朝那个方向走。快是快，但可能走进死胡同，只盯着眼前看起来最近的路。

---

### 1.2 A\* 算法

#### 核心公式

**f(n) = g(n) + h(n)**

| 符号 | 含义 | 大白话 |
|------|------|--------|
| **g(n)** | 从起点到当前节点累计的实际代价 | 已经走了多少路 |
| **h(n)** | 从当前节点到终点的预估代价（启发信息） | 到终点大概还要走多少 |
| **f(n)** | 总体代价值 | 这条路总共划不划算 |

> 💭 **我的理解：**
> - A* 其实就是给"贪心搜索"加了一个 **已经走了多少** 的约束，防止只顾方向不顾成本
> - `h(n)` 是"猜的"，不是真实距离——猜得越准，搜得越快
> - 拿 Arad → Bucharest 举例：`h(n)` = 直线距离，`g(n)` = 实际走过路线长度，A* 就是两者一起算，选最划算的那条

#### 三种算法对比

| 算法 | f(n) | 一句话 |
|------|------|--------|
| **Dijkstra / UCS** | g(n) | 只管走过多远，不看目标，稳但慢 |
| **贪心最佳优先** | h(n) | 只看目标方向，快但可能走偏 |
| **A\*** | g(n) + h(n) | 两者兼顾，又快又稳 |

| 类比 | 算法 |
|------|------|
| 全图地毯式搜索 | Dijkstra |
| 只盯方向不看路况 | 贪心最佳优先 |
| 智能导航，方向 + 成本一起算 | **A\*** |

> 💭 **我的理解：** A* 不是分别优化了 Dijkstra 和贪心，而是把它们统一到了 `f(n) = g(n) + h(n)` 一个框架里——当 `h(n) = 0` 就退化为 Dijkstra，当 `g(n) = 0` 就退化为贪心。

#### 最优性条件

- A* 保证最优解的前提：**h(n) 是可采纳的（admissible）**
- 即 `h(n)` 永远不会高估实际代价（如用直线距离估计实际路程）
- 常用启发函数：曼哈顿距离、欧几里得距离

#### 历史脉络

```
Dijkstra (1956)  →  贪心最佳优先  →  A* (1968) 结合两者
```

## 二、对抗搜索（博弈搜索）

> **场景：** 双人零和博弈（棋类游戏）——你赢就是我输，双方轮流出招

### 2.1 最小最大搜索 (Minimax)

- 双方轮流走棋，构建**博弈树**
- **MAX（我方）：** 选分数最大的走法
- **MIN（对手）：** 选对我方最不利（分数最小）的走法
- 双方都假设对方会走最优一步，递归评估到终局

> 💭 **我的理解：** 下棋无非是"我走一步，对手走一步，我再走一步..."——每一步都要想"如果我这么走，对手肯定会走最狠的那步，那我得选一个对手搞完我之后我还能赢的"。这就是 Minimax 的全部逻辑。

#### 问题

- 围棋分支因子 ~250，全算完不现实

---

### 2.2 阿尔法贝塔剪枝 (Alpha-Beta Pruning)

Minimax 蠢在每步都算。剪枝的思路：

> "这条分支你已经知道不会走了，还算它干嘛？"

- **α（alpha）：** MAX 方"至少能拿多少分"的下限，初始 -∞
- **β（beta）：** MIN 方"最多能给多少分"的上限，初始 +∞
- 剪枝条件：当分支评估值 ≤ α 或 ≥ β 时，直接剪掉

> 💭 **我的理解：** α-β 就是 Minimax 的"减负版"——把明显不会被选的分支砍掉。搜索量从 O(b^m) 降到 O(b^(m/2))，同样的时间能多搜一倍的深度。本质上算的还是同一套逻辑，只是聪明地跳过无用分支。

#### 优化方向

- 优先评估更有希望的分支，触发更多剪枝
- 缓存已评估状态（置换表）
- 结合迭代加深搜索

---

### 2.3 三种算法对比总结

| 算法 | 思路 | 痛点 |
|------|------|------|
| Minimax | 全算一遍 | 太慢，围棋根本算不完 |
| α-β 剪枝 | 砍掉没用的分支 | 仍需评估函数判断局面 |
| MCTS | 不穷举，靠随机模拟评估节点 | 需要模拟次数够多 |

> 💭 **我的理解：** 这是个递进优化的故事：
> - Minimax 算不完 → α-β 砍分支省计算 → MCTS 干脆不穷举，靠摸奖式模拟
> - 围棋走法实在太多，Minimax 和 α-β 都搞不定，所以 AlphaGo 用的是 MCTS + 深度学习

#### 什么时候用什么

| 场景 | 算法 | 原因 |
|------|------|------|
| 井字棋 / 简单残局 | Minimax | 状态空间小 |
| 国际象棋 | α-β + 评估函数 | 分支多但有好的评估函数（深蓝） |
| 围棋 | MCTS + 深度学习 | 状态空间太大，评估函数难写（AlphaGo） |

## 三、蒙特卡洛树搜索 (MCTS)

> 与 Minimax / α-β 不同，MCTS **不需要评估函数**，通过随机模拟评估节点价值。是 AlphaGo 的核心算法。

### 3.1 理论基础：多臂赌博机

- 多台老虎机，每台胜率未知——如何最大化收益？
- 核心矛盾：**探索 vs 利用**
  - **利用：** 一直玩目前胜率最高的
  - **探索：** 试试其他机器，也许更好

### 3.2 UCB 策略

$$UCB = \underbrace{\frac{w_i}{n_i}}_{\text{利用}} + \underbrace{c \sqrt{\frac{\ln N}{n_i}}}_{\text{探索}}$$

| 符号 | 含义 |
|------|------|
| $w_i$ | 节点 i 的获胜次数 |
| $n_i$ | 节点 i 的访问次数 |
| $N$ | 父节点总访问次数 |
| $c$ | 探索因子（控制探索/利用权重） |

> 💭 **我的理解：** UCB 公式就是"既要又要"——胜率高当然多选，但如果一条路试太多遍还没新路试过，就优先试试新路。这个平衡是 MCTS 不卡在局部最优的关键。

### 3.3 四个核心步骤

```
选择 (Select)  →  扩展 (Expand)  →  模拟 (Simulate)  →  回溯 (Backpropagate)
    │                   │                   │                    │
用 UCB 从根节点    到了未完全展开     从叶节点开始随机     把模拟结果沿路径
选最优子节点，     的节点，新建一     走棋直到终局         向上更新所有节点
一路走到叶节点     个子节点           (Rollout)            的胜率和访问次数
```

> 💭 **我的理解：** 本质上就是四个字——**少想多试**。不看死了每一步怎么走，而是模拟几千几万局让结果说话。哪条路胜率高就走哪条。

### 3.4 时间复杂度

- 没有固定公式，由模拟次数和深度决定
- 不像 Minimax 受限于 $b^m$，可以随时停止
- 模拟越多结果越可靠

### 3.5 局限性

> 💭 **我的担心：** 万一自己觉得有前途的路其实都不太行？那模拟不就是在"自欺欺人"吗？

UCB 公式的**探索项**天然防御这个问题：

- 一条路试多了，$n_i$ 变大，探索项变小 → **自动把算力让给没试过的路**
- 不会永远卡在"自以为好"的路上

**但这个保险不是万能的：**

| 局限 | 说明 |
|------|------|
| 首次误导 | 开局某烂走法运气好连赢几局，UCB 短期偏袒，要大量模拟才能纠正 |
| 稀疏奖励 | 围棋到最后才知道输赢，早期模拟结果参考价值低 |
| Rollout 太粗糙 | 随机乱下和高手走棋差距大，容易误判 |

**优化方向：**

- **调探索因子 c** — 前期大（多探索），后期小（多利用），类似模拟退火的温度递减
- **上神经网络** — AlphaGo 不用随机 Rollout，用神经网络预测走法和局面好坏
- **逐步剪枝** — 模拟一小段就评估，不好就停，不浪费算力

---

## 速查卡片

- **A\*：** `f(n) = g(n) + h(n)`，h(n) 可采纳则最优
- **Minimax：** 我最优你最优，全算 → 太慢
- **α-β 剪枝：** 砍无用分支，O(b^(m/2)) → 仍需评估函数
- **MCTS：** 不穷举，随机模拟 → UCB 平衡探索/利用 → AlphaGo
- **深度学习 vs 搜索：** 搜索做决策，深度学习做预测，互补
